# Agents Weekly 备课 Space 设计

## 交付定义

建设一个长期公用的 Cohub Space，帮助 Agents Weekly 嘉宾用较少的额外负担完成高质量备课。Space 同时支持只有主题和已有讲稿两种入口，通过多个 Agent 并行反馈、统一修改建议和独立验收，降低课程节奏不当、内容发散、重点不突出和听众记不住的风险。

交付完成不以 Space 存在或 Agent 有返回为准，而以真实备课链路可走通为准。嘉宾能提交主题或已有材料，收到经过合并的反馈和可直接使用的修订结果，完成一轮修改，再由独立 Agent 给出通过或不通过的明确结论。

## 设计原则

嘉宾感受到的应该是备课搭档，不是填表和审核。Space 优先读取嘉宾已有文档、作品、案例和演示材料，能提取的信息不再重复询问。每次交互只要求嘉宾处理少量高价值问题，并立即返回更好的标题、主线、提纲或时间表。

质量标准是必过关卡，但任何不通过都必须附带具体原因和可采用的修改稿。Agent 不得因为文档完整、排版精美或嘉宾已经花费较多时间而降低标准。

## 嘉宾流程

嘉宾进入长期公用 Space 后，从零开始或带稿校准两种入口中选择一种。每次备课建立独立课程档案，保存原始输入、各视角反馈、统一修改稿、嘉宾确认稿和验收结果。

多个 Agent 同时完成一轮会诊。协调 Agent 不把原始反馈堆给嘉宾，而是消除重复和冲突，只保留最需修改的三件事，同时产出修订后的课程主线、内容结构和时间表。嘉宾可以直接接受、局部调整或说明不接受的原因。

嘉宾确认后，独立验收 Agent 只读取课程目标、最终材料和质量标准，不继承前面的辩解和改稿过程。验收不通过时，只处理剩余的阻塞问题，不让嘉宾重走整条流程。

有稿嘉宾的主动备课时间目标不超过三十分钟，从零开始的目标不超过四十五分钟，不包含嘉宾自己完成作品或演示的时间。

## Agent 角色

听众 Agent 以第一次接触主题的参会者身份阅读，找出听不懂、容易跟丢和无法复述的位置。它必须给出自己实际记住的主线和收获，不得只作文字润色。

内容编辑 Agent 负责找出课程的唯一主线，删掉并列主题和无法服务主线的材料，把核心收获控制在三个以内，并明确哪些内容值得展开。

现场导演 Agent 负责三十到六十分钟分享和五到十分钟问答的节奏。它检查开场、转场、案例、演示、总结和问答预留，发现内容超载时必须明确删减。

质疑 Agent 负责检查关键结论是否有真实案例、演示或可检查的证据支撑，并准备参会者最可能追问的问题。

协调 Agent 负责合并结果，而不参与投票。当不同 Agent 意见冲突时，它依据课程目标、听众理解和时间限制取舍，并将取舍理由留在课程档案中。

验收 Agent 负责最终关卡，不得被前置 Agent 的通过判断影响。它只能给出已通过或需修改，需修改时必须标出阻塞放行的具体问题。

## 课程产物

每门课程最终产出一份可上场使用的课程包。课程包包含标题、一句话主线、三个以内的核心收获、按分钟分配的提纲、案例或演示清单、关键转场提示、预计问答和三道会后回忆题。课程包是嘉宾备课的实际成果，不需要嘉宾再把 Agent 的评论整理成稿件。

每个课程档案有准备中、待确认、需修改和已通过四种状态。只有已通过的课程包能进入公用案例库。原始材料、未通过版本和审核记录保留在课程档案中，不得被自动覆盖。

## 通过标准

课程必须只有一条可复述的主线，核心收获不超过三个，并在开场三分钟内说明主题与听众的关系。每个核心收获至少有一个案例、演示或具体证据支撑。不能被新手听众理解的术语必须解释或删除。

课程时间表必须落在选定的三十到六十分钟分享时间内，另行留出五到十分钟问答，不允许通过压缩问答来掩盖课程超载。按提纲进行的时间模拟不得超过目标时长的百分之十。

验收 Agent 以陌生听众身份阅读完最终材料后，必须能用自己的话复述主线和至少两个核心收获。如果它只能重复标题，或复述结果与嘉宾设定的重点不一致，课程不得通过。

## 会后反馈

每次分享结束后发布三道轻量回忆题，不用满意度代替理解检查。回忆题在次日收集，结果回写课程档案。至少百分之七十的作答者答对两道，才表明课程的记忆目标达成。未达成时保留真实结果，标记听众实际丢失的重点，作为后续课程的证据，不回头篡改已发生课程的通过状态。

## 失败处理

输入材料无法读取、关键案例缺失、内容真实性无法确认或剩余时间不足以完成必要修改时，Space 必须保留当前材料并明确标记阻塞原因。它不得把缺失材料当作空值跳过，不得用生成的假案例补齐，不得因为周会即将开始而自动放行。

不同 Agent 反馈冲突时，协调 Agent 必须展示它选择保留的判断及依据。无法根据课程目标、听众理解和时间限制解决的冲突，由嘉宾做内容选择，但不能豁免最终验收。

## 对抗性验收

正式开放前，必须用四类故意制造的坏输入实测。只有主题的输入必须得到一份可讲的课程包，而不是通用建议。排版精美但主线发散的长稿必须被拦下并重组。明显超时的提纲必须给出具体删减，不能只建议讲快一些。只有观点没有案例的材料必须保持需修改状态，不得自行虚构证据。

验收还需检查反馈负担。协调后的必改项不得超过三个，不得保留相互矛盾的建议，每个问题都必须附带可直接采用的修改结果。嘉宾完成一轮修改后，只要未出现新的阻塞问题，系统就应进入独立验收，不得无限追加新建议。

## 验收失败判据

任一故意制造的坏输入被错误放行，就不得宣布 Space 可用。嘉宾必须阅读多份重复或冲突的 Agent 报告，就不符合低负担目标。最终课程包没有明确主线、时间表、案例安排或问答准备，就不算完成备课。任何因为超时、缺材料或执行出错而生成的默认通过状态，都是阻断交付的故障。

## 范围边界

第一版只解决 Agents Weekly 的备课与放行，不建设通用在线课程平台，不代替嘉宾制作专业作品和现场演示，不自动发布课程，不把满意度当作课程质量的主要依据。
