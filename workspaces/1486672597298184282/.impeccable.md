## Design Context

### Users
AutoSprite is for game developers, technical artists, and solo creators who need to turn a single character image into usable spritesheet exports quickly. The product is used in an execution context, not a browsing context: people come in to upload a character, tune motion, preview results, and export files without breaking flow.

### Brand Personality
Focused, cinematic, and craft-aware. The product should feel like a premium game-tool workbench with a little showmanship, not a generic admin dashboard. It should communicate confidence and speed while still respecting the creator's art.

### Aesthetic Direction
Use the public AutoSprite dusk language as the anchor: charcoal surfaces, warm amber accents, cool blue atmospheric glows, rounded shell geometry, serif display typography, and clean sans body copy. Translate that language into a task-first workbench instead of a landing page. Avoid beige notebooks, generic white SaaS cards, glassmorphism spam, or purple AI gradients.

### Design Principles
Keep the first screen legible and useful, with the active character, current state, and next actions visible immediately.
Use one strong visual hierarchy: selected character first, build controls second, exports third.
Make dark surfaces feel intentional through contrast, typography, and spacing, not through heavy decoration.
Every panel should feel like a tool surface with a single job.
Keep the product shippable on desktop and mobile without hiding core actions.
Prefer staged flows over long mixed workbenches when the product is asking for sequential decisions.
Keep shell stable while changing the center task, so the user always knows context, action, and consequence.
